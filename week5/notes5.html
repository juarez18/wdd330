Debugging

The first thing is understanding the 3 types of errors: System, Programmer and User error. System errors represent a problem with the system or devices the program is interacting with. A program error is the problem with the program syntax or logic or user error from the user breaking the program in some way. An Exception is an error that returns a value the program can be used by the program to deal with the error like referencing a variable that does not exist. Stack traces help find the origin of a problem with a program. A warning is not a fatal issue to your program but it can be a problem when it runs and is something that you should definitely take steps to resolve. You can use ‘use strict’ to declare in functions so that you can keep good coding habits and catch more errors. You can use an If statement to check that a feature exists and then call a function to use an API object.

Testing

Test-Driven development(TDD) is writing tests before any actual code this will help in development as it makes your code stronger and less likely to fail as you develop it. This is the workflow of TDD:

  1. Write tests.
  2. Write code to pass the tests.
  3. Refactor the code.
  4. Test refactored code.
  5. Write more tests for new features.

There are many frameworks that can lighten the load of writing endless tests. One that is common is Jest this was created by Facebook and is easy to use in a variety of ways.
